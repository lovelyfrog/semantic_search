# Semantic Search Skill

你可以通过执行 `semantic-search` 命令行工具，对当前代码仓库进行语义索引与检索。

`SEMANTIC_SEARCH_PROJECT` 环境变量已指向当前仓库根目录，所有命令无需额外传入 `--project`。

---

## 索引

```bash
semantic-search index --layer all --output json
```

- **同步阻塞**：命令运行直到索引完成后返回，无需轮询进度。
- 输出 JSON，包含索引状态与处理的文件/符号数量。

**何时需要执行索引：**
- 首次在当前仓库执行 `search` 之前
- 仓库代码发生大规模变更后
- `search` 返回结果明显滞后或为空时

---

## 搜索

```bash
semantic-search search \
  --query "<查询文本>" \
  --layer symbol \
  --limit 10 \
  --threshold 0.5 \
  --output json
```

**`layer` 选择：**

| 场景 | `--layer` |
|------|-----------|
| 定位函数/类/接口/符号定义（默认） | `symbol` |
| 找哪个文件最相关 | `file` |
| 读文档/注释/README/协议文本 | `content` |
| 需要综合候选集 | `all` |

**参数调整建议：**
- `--limit`：精准定位时可降至 5；结果较少时可提高到 20
- `--threshold`：不建议低于 0.4（噪音多）或高于 0.8（漏结果）
- `--path src/foo`：用户明确限定目录/模块时追加，可重复传入

---

## 何时调用 `search`

当用户问题满足以下任一条件时，**优先执行 `search`**，而非直接读取文件或全仓扫描：

- **定位类**："X 在哪里实现/定义？""某接口/函数/struct 在哪个文件里？"
- **关系类**："谁调用了 X？""X 会触发哪些逻辑？""这段逻辑和哪些模块相关？"
- **意图类（不知道精确关键词）**："权限校验在哪里做的？""错误处理逻辑怎么运作？"
- **跨文件/跨模块**：需要从多个位置快速收敛候选范围

**不适合用语义搜索：**
- 查找字符串字面量的出现位置 → 优先用 ripgrep / grep
- 纯概念解释、不依赖代码细节的问题

---

## 行为准则

```
当用户在问"代码在哪里/怎么实现/谁调用谁/相关模块有哪些"，执行 search 命令。
如果 search 返回空结果或结果明显不准确，先执行 index 再重试 search。
首次在新仓库使用时，先执行 index，再执行 search。
```
